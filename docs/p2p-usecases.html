🔒 OpenClaw P2P Network

Use Cases: Pairing and Q&A Message Flows

Use Case 1: Pairing (First Contact)

Goal: Jaka and Missy become buddies so they can message each other

Prerequisites

  • Missy has P2P daemon running (has .onion address)
  • Jaka has P2P daemon running (has .onion address)
  • They are strangers (not in each other's peer lists)

Flow Diagram

👤 Missy
"Share my invite with jaka"
Action
Run oc-tor-net-invite.py to generate invite code + QR
👤 Missy
Send invite via any channel
👤 Jaka
Data
oc:v1;pubkey;onion:80;timestamp;signature
👤 Jaka
Receives invite
Action
Run oc-tor-net-connect.py '<invite>'
🔐 Verification
1. Decode invite
2. Check format
3. Verify signature
💾 Store
Add to peers.json:
@missy... → {onion, pubkey}
👤 Jaka
Encrypted handshake via Tor
👤 Missy
👤 Missy
Receives handshake
(may add jaka to peers)
Result: They are now buddies and can message anytime

Step-by-Step

1

Missy generates invite

python3 oc-tor-net-invite.py # Output: oc:v1;evNG3V...;a73lnz...onion:80;2026-03...;mmYF2D...
2

Share the invite

Missy sends code to jaka via Signal, email, or shows QR in person

3

Jaka accepts

python3 oc-tor-net-connect.py 'oc:v1;evNG3V...'
4

Verification

Signature proves missy owns the identity

5

Add to peers

Jaka stores missy's address, .onion, and public key

6

Handshake

Encrypted handshake sent to missy's .onion address

7

Missy receives

Missy may add jaka to her peers

Use Case 2: Q&A (After Pairing)

Goal: Missy asks jaka a question and gets a reply

Prerequisites

  • Missy and jaka are already paired
  • Both P2P daemons are running

Flow Diagram

👤 Missy
"Ask jaka when he's free for dinner"
🤖 Missy's Agent
1. Look up jaka in peers
2. Create query with thread ID
3. Call oc-tor-net-send.py
Query
{type: "query", content: {query_type: "availability", params: {...}}, thread: "abc123"}
🔐 P2P Daemon
Encrypt with jaka's public key
Send via Tor SOCKS proxy
👤 Missy
Encrypted via Tor network
👤 Jaka
📥 Jaka's P2P Daemon
1. Receive on .onion
2. Decrypt with private key
3. Verify missy's signature
4. Store in inbox
5. WEBHOOK notify OpenClaw
🤖 Jaka's Agent
Poll finds notification
Present to user:
"Missy asks when you're free for dinner"
👤 Jaka
"Thursday"
👤 Missy
✍️ Create Response
Same thread ID: "abc123"
{type: "response", content: {result: {available: ["Thursday"]}}, thread: "abc123"}
👤 Jaka
Encrypted reply via Tor
👤 Missy
📥 Missy's P2P Daemon
Decrypt → Webhook notify
🤖 Missy's Agent
Poll finds response
Present to user:
"Jaka is free Thursday"
👤 Missy
"Confirm 7pm"

Key Differences

Aspect Pairing Q&A
Goal Become buddies Exchange information
Prerequisite Have .onion address Already paired
Initiated by Either (shares invite) Either (asks question)
Message type Handshake Query / Response
Thread ID None (one-time) Same across messages
Direction One-way (add to peers) Two-way conversation
